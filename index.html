Git/GitHub

¿Qué es Git?

Git es un sistema de control de versiones distribuido y de código abierto, diseñado para manejar todo, desde proyectos pequeños hasta grandes con velocidad y eficiencia. Fue creado por Linus Torvalds en 2005 para desarrollar el kernel de Linux. Las principales características de Git incluyen:

  • Ramas (branches): Permiten trabajar en diferentes versiones de un proyecto simultáneamente.
  • Commits: Guardan cambios en el repositorio, permitiendo rastrear y revertir cambios.
  • Repositorio (repository): Es el lugar donde se almacena el historial completo de cambios de un proyecto.
  • Fusión (merge): Combina diferentes ramas en una sola.
  • Clonación (clone): Permite copiar un repositorio completo para trabajar en él localmente.

¿Qué es GitHub?

GitHub es una plataforma de desarrollo colaborativo que utiliza Git para el control de versiones. Además, ofrece herramientas para facilitar la colaboración entre desarrolladores, como:

  • Repositorios remotos: Almacenan el código en la nube, accesible para múltiples usuarios.
  • Issues y Pull Requests: Facilitan la colaboración y revisión de código.
  • Integración Continua: Permite automatizar pruebas y despliegues.
  • Comunidades y Equipos: Fomenta la colaboración entre desarrolladores a través de organizaciones y equipos.

¿Cómo funciona GitHub Pages?

GitHub Pages es un servicio de GitHub que permite alojar sitios web estáticos directamente desde un repositorio de GitHub. Es ideal para alojar páginas personales, blogs, o sitios web de proyectos. Funciona de la siguiente manera:

  1. Repositorio: Crea un repositorio en GitHub.
  2. Contenido: Añade archivos HTML, CSS, y JavaScript al repositorio.
  3. Configuración: En las opciones del repositorio, habilita GitHub Pages.
  4. Despliegue: GitHub Pages generará automáticamente una URL donde estará alojado tu sitio web (por ejemplo, https://tuusuario.github.io/turepositorio).
  5. Actualizaciones: Cada vez que hagas un push a la rama configurada para GitHub Pages (main o gh-pages), el sitio se actualizará automáticamente.

Box Modeling

Forma de pensar la estructura para realizar diseños basada en cajas.

  • Propiedades de la caja (cómo se forma la caja)
    • width
    • height
    • padding
    • margin
    • border
  • Forma en que se calcula el tamaño de la caja
    • content-box: El espacio final ocupado por la caja es el resultado de la suma de su contenido, width, padding, margin, border
    • border-box: El espacio final ocupado por la caja es el width
  • Características de la caja (tipo de display)
    • Elementos de bloque (display: block) - El elemento ocupa el 100% del ancho de su contenedor, fuerza salto de línea y se le puede cambiar ancho y alto
    • Elementos de línea (display: inline) - El elemento ocupa solo el tamaño de su contenido, no fuerza salto de línea y no se le puede cambiar ancho y alto
    • Elementos de línea-bloque (display: inline-block) - El elemento ocupa solo el tamaño de su contenido, no fuerza salto de línea y se le puede cambiar ancho y alto
    • Flexbox (display: flex) - El elemento tiene la capacidad de indicarle a sus hijos cómo deben ubicarse. Sólo puede hacerlo de forma horizontal o vertical

Tipos de medida

Herramienta que me permite declarar tamaño. Un tipo de medida le indica al navegador que cuenta tiene que hacer para conocer el tamaño

  • Medidas relativas
    • Porcentaje (%) - tamaño = ancho/alto del elemento contenedor directo * %
    • em - tamaño = medida de tipografía del elemento padre * em. El tamaño final se ve afectado por el tamaño de la tipografía de todos los elementos contenedores
    • rem - tamaño = medida de tipografía del navegador * rem
    • viewport(vw, vh) - tamaño = ancho/alto de la pantalla * vw%/vh%
      • width: 50vw; - tamaño = 50% del ancho de la pantalla
      • height: 50vh; - tamaño = 50% del alto de la pantalla
  • Medidas absolutas - tamaño = valor otorgado
    • px

Flexbox

Flexbox es una herramienta que nos permite manipular el contenido de un elemento HTML. Es un modelo de diseño de cajas en el que los elementos se acomodan en líneas horizontales o verticales.

Para poder ubicar a los elementos, el navegador necesita conocer los tamaños. Por ejemplo:

Propiedades de Flexbox

  • align-items - Ubica a los elementos en el eje secundario

    Valores

    • center
    • flex-start
    • flex-end
    • align-self - Se aplica al elemento hijo y permite moverlo de forma "independiente" del padre. Solo sirve si el elemento padre tiene display: flex
  • justify-content - Ubica a los elementos en el eje principal

    Valores

    • center
    • flex-start
    • flex-end
    • space-evenly - El navegador deja espacios equitativos a todos los lados de los elementos
    • space-around - El navegador deja el doble de espacio en los elementos internos y los externos los separa de forma equitativa
    • space-between - Los elementos externos se pegan a los bordes del contenedor padre
  • gap: Permite declarar un espacio entre elementos
  • flex-wrap: Permite indicar qué hacer cuando los hijos ya no entran. Por defecto, el elemento hijo se achica

Ejemplos

flex-direction: flow

Con flex-direction: flow ubicamos los elementos de forma horizontal. El eje principal es el eje X

flex-direction: column

Con flex-direction: column ubicamos los elementos de forma vertical. El eje principal es el eje Y

Grid

Un Grid es un diseño flexible y móvil que divide el contenido en celdas en un contenedor. El contenido se organiza en líneas y columnas en función de las propiedades de ancho y alto de las celdas.

Propiedades de declaración de filas y columnas

Una fila/columna se declara indicando cuál es el tamaño que debe ocupar. El tamaño de las filas/columnas debe pensarse en relación al tamaño del elemento que tiene display grid

  • grid-template-columns - Permite declarar columnas
  • grid-template-row - Permite declarar columnas

Tipo de medida fr

Medida propia de las grillas que permite la distribución del espacio ocupado por las filas y columnas sin sobrepasar el tamaño del contenedor padre

Propiedades de ubicación de la grilla

Sólo tiene sentido utilizar estas propiedades si la grilla no ocupa el espacio total del elemento contenedor

  • justify-content - ubica la grilla en el eje x
  • align-content - ubica la grilla en el eje y

Propiedades para indicar cuánto espacio ocupa un elemento

Todos los elementos se posicionan en el primer cuadrante libre. Siempre se empieza a contar desde el cuadrante de arriba a la izquiera. Todos los elementos se ubican uno detras del otro. Los elementos se posicionan por orden de lectura del HTML

Position

La propiedad position permite controlar la ubicación de un elemento en el documento. Tiene cuatro valores posibles: static, relative, absolute, fixed y sticky.

Todo elemento que tenga en su propiedad position un valor distinto de static va a permitir el trabajo en capas ("apilar cajas hacia afuera")

  • static (default) - La caja ocupa espacio. El espacio ocupado dependerá de su propiedad display y de su elemento contenedor
  • relative - La caja ocupa el espacio que tiene asignado pero puede moverse y apilarse. Se usa como referencia para otros elementos con position distinto de static
  • absolute - La caja no ocupa el espacio asignado, por el contrario se posiciona en relación al primer elmento padre que no tenga position static o, si ninguno de sus elementos padre tiene un position distinto de static, se posiciona en relación al body
  • fixed - La caja no ocupa el espacio asignado. Se ubica en relación a la pantalla
  • sticky - La caja ocupa el espacio que tiene asignado, puede moverse y "se pega" al borde declarado.

Mover elementos con position distinto de static

Para mover un elemento con position distinto de static, se utilizan las propiedades top, bottom, left, y right. Estas propiedades se utilizan para indicar la posición en píxeles del documento o en relación al elemento padre que tiene position relative.

Si utilizo top: 10px. El elemento se mueve desde arriba 10px hacia abajo

Si utilizo left: 10px. El elemento se mueve desde la izquierda 10px hacia la derecha

Si utilizo top: -10px. El elemento se mueve desde abajo 10px hacia arriba

Si utilizo left: -10px. El elemento se mueve desde la derecha 10px hacia la izquierda

Modificar el orden de las capas

La propiedad z-index permite indicar qué caja debe ubicar más arriba y cuál más abajo. Cuanto más alto el valor de z-index, más arriba se ubica la caja
